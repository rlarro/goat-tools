# End-of-Session Checklist

**Purpose:** Run this after every chapter lock. It keeps your project files in sync and prevents the accumulated drift that makes later sessions harder. Every item is here because it was missed at least once.

**Important:** Do NOT try to do all steps in one pass. Each step requires reading the current version of a file, making changes, and outputting a new versioned file. Do them one at a time.

---

## Before you start

- Read this entire checklist before beginning any step.
- For each file, find the most recent version in your project files (look for the highest version suffix).
- Increment the version number for each file you update.
- Present each file when done. Wait for confirmation before moving to the next step.

---

## Step 1: Update the Manuscript

Find the most recent manuscript file.

- Insert the new locked chapter(s) with proper heading format
- Apply any corrections to earlier chapters noted during the session
- Update the table of contents (add checkmarks for newly locked chapters)
- Clean any encoding artifacts (escaped quotes, stray formatting)
- Verify chapter headers
- Increment the version number

---

## Step 2: Update the Style Guide

Find the most recent style guide.

- Update the date
- Update chapter map entries (mark newly locked chapters)
- Propagate any title changes
- Resolve any open items that were decided this session
- Add any new open items
- Add any new character names to the names section
- Add any new editorial principles discovered during the session
- Update callback tracking if threads were planted or landed

---

## Step 3: Update Any Project-Specific Tracking Documents

[Customize this step for your project. Examples:]

- Chapter log with word counts and per-chapter decisions
- Raw material / notes file (add new material, move spent material to archive)
- AI collaboration log (if you're tracking your process)

---

## Step 4: Write the Session Handoff

- Write the opening prompt for the next chapter session
- Include: what to read, what's locked, what the next chapter inherits, scope and key beats, any sensitive material notes
- Save as your session handoff file (overwrite the previous one)

---

## Step 5: Final File List

After all steps are complete, list every file that was updated or created with its new version number. Confirm the list so nothing gets missed when saving.

---

## Session-Specific Notes

Before starting the checklist, list any session-specific items that need to be captured:

- Corrections to earlier chapters
- Title changes to propagate
- New project files created during the session
- Deferred decisions for future chapters

---

## How to use this

1. **Run it every time.** Skipping "just this once" is how project files drift. By the time you notice, you've lost track of what's current.

2. **The sequential approach exists for a reason.** Trying to update everything at once hits tool limits and causes mistakes. One file at a time.

3. **Customize Steps 3 and 4.** Every project has different tracking documents. Add or remove steps based on what your project actually uses.

4. **This checklist will grow.** When you discover a step you keep forgetting, add it here. The checklist is a living document.

---

## Real Example: End-of-Session Checklist from *Too Many Goats on the Mountain*

The TMG project ran for 100+ sessions across 34 chapters. By the end, the checklist had seven steps:

1. **Update the Manuscript.** Insert locked chapter, update TOC, clean encoding artifacts, verify headers, increment version. (The manuscript went through 41 versions.)
2. **Update the Style Guide.** Update chapter map, resolve open items, add new names, log new editorial principles. (The style guide went through 9 versions.)
3. **Update the Chapter Log.** Word counts, per-chapter decisions, voice lessons, process lessons. (A separate tracking file from the style guide.)
4. **Update Random Thoughts.** Add new raw material, move spent material to archive. (The parking lot for future chapters.)
5. **Update the AI Collaboration Log.** Timeline entries, process milestones, voice lessons, structural decisions. (This became the raw material for the white paper about the project.)
6. **Write the Chapter Opening Prompt.** The session handoff for the next chapter, including what to read, what the next chapter inherits, and any sensitive material notes.
7. **Final File List.** List every file updated with its new version number. Confirm nothing was missed.

### What makes this example work

Every step specifies a single file. The sequential approach (one file at a time, confirmed before moving on) exists because trying to do everything at once hit tool limits and caused mistakes. Step 5 (the AI Collaboration Log) was unique to this project since it was tracking the process itself. Step 6 exists because without it, the next session would spend its first thirty minutes rebuilding context that should have been captured while it was fresh.
